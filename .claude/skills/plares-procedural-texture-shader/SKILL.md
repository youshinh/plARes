# Procedural Texture & Shader Expert

This skill guides the mapping of dynamically generated backend images (from Nano Banana Pro) onto frontend 3D models.

## When to Use This Skill

- When the backend returns a URL containing a newly generated texture (e.g., evolution skin, damage decals).
- When writing raw GLSL shaders to mix multiple maps (base color + damage mask).

## Instructions

1. **Dynamic Loading**:
   - Use `THREE.TextureLoader` to asynchronously load image URLs generated by the backend.
2. **Material Update**:
   - Apply the loaded texture to `mesh.material.map` (or `roughnessMap`, `normalMap`).
   - You MUST call `material.needsUpdate = true` for Three.js to recompile the shader.
3. **Shader Manipulation (Advanced)**:
   - Use `onBeforeCompile` or custom WebGL `RawShaderMaterial` to mix the damage/evolution map over the original texture dynamically based on a uniform (e.g., `healthRatio`).

## Examples

### Loading AI Generated Texture

```javascript
function applyGeneratedSkin(mesh, textureUrl) {
  const loader = new THREE.TextureLoader();
  loader.crossOrigin = "anonymous";

  loader.load(textureUrl, (texture) => {
    // Preserve existing color, replace map
    mesh.material.map = texture;
    mesh.material.needsUpdate = true;

    // Optional: play a glowing effect via Tween to signal the change
    playEvolutionVFX(mesh);
  });
}
```
