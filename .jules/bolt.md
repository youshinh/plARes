## 2024-05-14 - React Three Fiber traversal caching with async children
**Learning:** In Three.js/React Three Fiber, caching descendants (like meshes or materials) inside a `useEffect` to avoid per-frame `group.traverse()` in `useFrame` is a critical optimization. However, depending solely on a prop array like `attachments` is unsafe because 3D asset loading is asynchronous (via `GLTFLoader`). The prop changes immediately, but the meshes are added later, resulting in a stale cache that misses the dynamically loaded children.
**Action:** Always trigger the cache rebuild after the async loading *and* attachment steps are complete. One robust pattern is having the component/hook that manages the async attachment return an incrementing `version` number (via `useState`) each time an `add()` occurs, and using that `version` in the caching `useEffect` dependency array.
