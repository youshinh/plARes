## 2024-03-20 - Caching Three.js Materials to Avoid `useFrame` Traversals
**Learning:** Calling scene graph traversals (like `group.traverse()`) inside `useFrame` callbacks causes severe performance bottlenecks in React Three Fiber, especially with complex GLTF models. Furthermore, because 3D asset loading is asynchronous (e.g., via `GLTFLoader`), you cannot rely solely on the initial load state. If attachments or children are loaded dynamically, caching materials too early results in a stale cache that misses dynamically loaded meshes.
**Action:** Always cache target materials or meshes in a flat array (`useRef`), and populate this cache inside a `useEffect` hook. To ensure the cache stays up-to-date with dynamically loaded children, trigger rebuilds by passing an incrementing `version` number (returned via `useState` from the attachment/asset manager each time an async `add()` occurs) into the `useEffect`'s dependency array.
