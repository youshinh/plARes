## 2026-03-02 - Prevent Path Traversal in File Operations\n**Vulnerability:** User IDs and Room IDs extracted from websocket URLs were passed unvalidated to `Path(...)` constructions, allowing arbitrary file system access.\n**Learning:** Implicit trust of WebSocket URL queries (`user_id`, `room_id`) led to dangerous string interpolation in filesystem paths.\n**Prevention:** Sanitize all identifiers used in filesystem paths utilizing a robust whitelist regex `re.sub(r'[^a-zA-Z0-9_-]', '_', val)` immediately prior to path construction.
## 2025-03-09 - Firestore NoSQL Path Traversal Vulnerability
**Vulnerability:** User-controlled strings (like `user_id` and generated `doc_id`) were interpolated directly into Firestore `document()` calls without sanitization.
**Learning:** Using untrusted inputs in database path references can allow malicious users to overwrite or access unauthorized collections by injecting `..` or `/` characters, leading to NoSQL Path Traversal.
**Prevention:** Always validate and sanitize inputs (e.g., using a regex-based `_sanitize_id` function that limits input to alphanumeric and safe characters) before passing them as document IDs to Firestore reference chains.
