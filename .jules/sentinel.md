## 2026-03-02 - Prevent Path Traversal in File Operations\n**Vulnerability:** User IDs and Room IDs extracted from websocket URLs were passed unvalidated to `Path(...)` constructions, allowing arbitrary file system access.\n**Learning:** Implicit trust of WebSocket URL queries (`user_id`, `room_id`) led to dangerous string interpolation in filesystem paths.\n**Prevention:** Sanitize all identifiers used in filesystem paths utilizing a robust whitelist regex `re.sub(r'[^a-zA-Z0-9_-]', '_', val)` immediately prior to path construction.
## 2024-05-24 - API Response Detail Leak via Python Exception Stringification
**Vulnerability:** External facing WebSocket connections (e.g. `LIVE_PATH`, `auth_token_create_failed`, `interaction_create_failed`) were exposing literal Python stacktrace/exception strings and internal hardcoded environment failure paths (`ADK_IMPORT_ERROR`) directly to the end user by passing `str(exc)` in JSON `detail` fields.
**Learning:** Broad exception handling blocks `except Exception as exc:` can inadvertently leak system details when their stringifications are embedded into API response payloads that are returned over the network.
**Prevention:** Explicitly decouple internal logging from external responses. Log the full `exc_info` internally via `logger.error`, and substitute `detail` payloads with sanitized, generic error strings for external consumers.
