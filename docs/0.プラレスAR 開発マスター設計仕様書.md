# プラレスAR 開発マスター設計仕様書

## 1. プロジェクト概要：「プラレスAR」とは

本プロジェクトは、現実空間（リビングの床や机の上、自作の木工作品など）を舞台に、自分だけのオリジナルAIロボット（プラレスラー）を召喚し、声とジェスチャーで指示を出して戦わせる「次世代WebAR×AIマルチエージェント・バトルゲーム」です。
単なる強い機体の殴り合いではなく、「競技性」と「モノづくり（DIY）」の喜びを融合させています。さらに、専用アプリのインストールを不要とし、URL共有のみで観客（オーディエンス）がブラウザ経由で乱入できるWebARベースの高度なエンタメシステムとして構築されます。

## 2. システム全体のハイブリッド・アーキテクチャ

アクションゲームに必須の「ゼロ遅延（競技性）」と、AIエージェントの「高度な推論・自律性」を両立させるため、システム全体を**「フロントエンド（体・感覚）」と「バックエンド（脳・戦略）」に完全に分離した非同期ハイブリッド設計**を採用しています。

システムは以下の4つのモジュールで構成されます。

- **モジュールA：フロントエンド（体・感覚）**
  - **技術スタック**: React, Three.js, WebXR Device API
  - **役割**: アプリ不要のWebAR環境を構築します。スマホカメラから現実空間の床や段差をスキャンし、Hit Test APIとDepth Sensing APIを用いてリアルタイムに3Dメッシュ化を行い、機体が移動可能なNavMesh（ナビゲーションメッシュ）を動的に生成します。また、通信ラグを嫌う移動や防御の基本操作は、ブラウザ標準の Web Speech API を用いてローカルで遅延ゼロで処理します。
- **モジュールB：バックエンドAI（脳・戦略）**
  - **技術スタック**: Python, Agent Development Kit (ADK) / Antigravity, Gemini 3.1 Pro (Multimodal Live API)
  - **役割**: スマホからの負荷処理を受け持ちます。セッションごとにスレッドセーフな非同期バッファである `LiveRequestQueue` を生成し、`asyncio.gather()`を用いたノンブロッキングな並行タスク制御により、映像（1〜2fps）と音声波形を継続的にストリーミングします。これにより、ユーザーからの割り込み（Interruption）をシームレスに処理し、滑舌判定や戦術指示（Function Calling）、実況生成などの推論を各エージェントとして常駐させます。
- **モジュールC：リアルタイム通信網（神経）**
  - **技術スタック**: WebRTC (Data Channel / Media Stream), WebSocket
  - **役割**: プロトコルを厳密に分離します。機体の座標や基本アクションの同期には、ラグを10〜30msに抑えるUDPベースの WebRTC Data Channel を用いたP2P通信を採用します。一方、AI向けの重いマルチメディア（音声・映像）転送には WebRTC Audio/Video Stream を、言語非依存のイベント同期（例：ダメージ通知）には WebSocket を利用します。
- **モジュールD：データ・記憶基盤（状態管理）**
  - **技術スタック**: Google Cloud Firestore, Firebase Authentication
  - **役割**: 試合中の高頻度なRead/WriteによるAPIコスト破綻と遅延を防ぐため、試合中のリアルタイムな状態管理（State Management）はADKのローカルメモリやセッション内でのみ保持します。試合終了時にのみ、AIにイベントを要約（Summarize）させ、Firestoreの長期記憶（aiMemorySummary）や試合履歴（matchLogs）としてUpdateします。

## 3. コア機能の実装要件と設計図

### 3.1. 3層優先度制御ステートマシン（FSM）

通信遅延によるカクつきを防ぎ、自然な自律挙動を実現するため、フロントエンドのレンダリングループ（`requestAnimationFrame`）内に、3層の優先度（Priority）制御を実装します。

1.  **Priority 1（高）ローカル音声の緊急割り込み**: Web Speech APIを用いて「避けろ！」などのプレイヤーの音声をローカルで検知し、機体の移動ベクトルに強力な物理インパルスを即座に加え、ラグゼロで回避させます（クラウド通信なし）。
2.  **Priority 2（中）AIからのマクロ戦術割り込み**: Gemini APIから非同期で送られてくるJSON形式の Function Calling（例：`{"action": "take_cover", "target_object": "obstacle_A"}`）を受信し、ステートと移動目標座標を強制的に上書きします。
3.  **Priority 3（低）ローカル基本ループ**: 平常時は、生成されたNavMesh上を経路探索（Pathfinding）アルゴリズムで毎フレーム計算し、自律的に移動させます。
    （詳細: `3.3層優先度制御ステートマシン設計書`）

### 3.2. モーション先行型のレイテンシ隠蔽と滑舌判定

AIの推論やAPI通信に伴うラグ（1〜2秒）をアクションゲームのUX上で完全に隠蔽する設計です。

- **レイテンシ・ハイディング**: プレイヤーが必殺技の発声トリガーを引いた瞬間、フロントエンドで即座に約3秒の「チャージアニメーション」を開始します。この描画と完全に並行して、バックエンドへ詠唱音声のストリーミングを行います。
- **Native Audio解析**: 送信された音声は、STT（テキスト化）をバイパスし、生の音声波形（16kHz PCM）のままGemini 3.1 ProのLive APIへ入力されます。滑舌判定エージェントが「正確さ」「スピード」に加え、波形から読み取れる「気迫（声の大きさや震え）」の3軸でスコアを算出し、アニメーション終了までにクリティカル判定のJSONをフロントエンドに返します。
  （詳細: `5.滑舌判定・レイテンシ隠蔽システム設計書`）

### 3.3. Geminiネイティブ機能のパラダイムシフト

LLMを単なるテキストジェネレータではなく、自律エージェントとしてフル活用します。

- **Context Caching**: 膨大な「過去の戦歴」や「物理ルール」をゲーム起動時に一度だけキャッシュし、入力トークンコストを劇的に削減しつつ、初動のトークン生成（TTFT）を爆速化します。
- **Managed MCP servers for Firestore**: 開発者が手動でデータベース検索APIを組むのではなく、AI自身にFirestoreへのクエリ実行権限（ツール）を付与します。これにより、AIが対話の文脈や現在の風景（Vision検知）から自律的に過去の記憶（aiMemorySummary）を検索し、「1ヶ月前のあの障害物の時、焦ってたよな」といった成長フィードバック（Proactive Audio）を自発的に生成します。
  （詳細: `6.プラレスAR：データ構造とMemory Bank統合設計書.md` および `7.プラレスAR：AI人格・ステータス進化設計書`）

### 3.4. マルチモーダル生成AIによる拡張エンタメ機能

観客の介入や報酬システムに、画像・動画・音楽生成APIを連携させます。

- **現実融合クラフト（Nano Banana Pro）**: 観客がスマホで撮影した現実のアイテム（例：夕食の春巻き）の写真と「巨大隕石」といったテキストプロンプトを受信し、Gemini 2.5 Flash Image（8-Image Mix機能）を用いて、1〜2秒で「現実の質感を残した3Dテクスチャやオブジェクト」を動的生成し、アリーナへ投下します。
- **マイルストーン動画・凱歌生成（Veo 3.1 / Lyria API）**: ダイナミック・ゲーティングの思想に基づき、試合終了時や5試合ごとのマイルストーンに限定して非同期バッチ処理を起動します。試合のテキストログとスクリーンショットをマルチモーダル入力とし、シネマティックなハイライト映像や、試合展開に応じたEDM等の勝利の凱歌を自動生成し、SNSへのシェア（バイラル拡散）を促進します。
  （詳細: `9.マルチモーダル生成AI・現実融合クラフト設計書`）

### 3.5. 言語非依存通信とServer-Driven UI

多言語対応は、静的な辞書ファイルではなく、ハイブリッド構成で実装します。

- 通信ペイロードには `{"event": "critical_hit", "user": "player1"}` といった言語非依存の軽量なJSONデータのみを使用し、戦術パネルや実況字幕はバックエンドAIが生成したJSONを受け取ってReactのStateにバインドし、動的に描画（Server-Driven UI）します。
- 試合終了後には、勝者のAIが相手の言語設定を読み取り、相手の母国語に自動翻訳して煽りや賞賛の「勝利者インタビュー」をGemini TTSでストリーミング再生するエモーショナル翻訳機能が稼働します。
  （詳細: `10.次世代プラレス：多言語対応・動的UI設計書`）
