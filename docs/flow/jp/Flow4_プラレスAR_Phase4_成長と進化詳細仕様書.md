# **プラレスAR ゲームフロー詳細仕様書**

## **Phase 4: 成長と進化（Memory Bank統合）**

[English Version (EN)](../Flow4_Phase4_Growth_and_Evolution_Detailed_Specifications.md)

本仕様書は、「プラレスAR」において対戦や日常の体験を「データ」として蓄積し、機体の視覚的な「外見（テクスチャ）」や内面的な「心（記憶と発話）」へと動的に還元する、フィードバック・サイクルのシステムフローと実装要件を定義する。

### **1\. ユーザーエクスペリエンス（UX）目標**

- **戦歴の視覚化（ワンオフの愛着）**: 傷一つ一つに理由がある、本物のプラモデルやDIY作品のような「世界に一つだけの使い込まれた機体」を自動生成する。
- **アンビエントな成長実感**: ステータス画面の数値を見せるのではなく、日常のふとした瞬間の「AIの独り言」によって、プレイヤー自身の成長や絆をエモーショナルに実感させる。
- **コストとUXの最適化**: 膨大な記憶をすべてAPIに都度読ませるのではなく、FirestoreとMCP、Context Cachingを組み合わせることでコスト破綻を防ぐ。

### **2\. 記憶の抽出とMemory Bank（データ層）**

試合中（Phase 3）の膨大な通信データを、永続的な「長期記憶」として圧縮・保存するバックエンド処理。

#### **2.1 インメモリからのログ抽出（Commit & Summarize）**

1. **試合終了トリガー**: 試合が終わった瞬間、バックエンド（Pythonサーバー）はメモリ上に保持していた数百行のアクションログ（移動、被弾、音声コマンドの波形評価など）をリスト化する。
2. **ハイライト抽出（Gemini API）**:
   - `gemini-3.1-pro-preview`にログを渡し、「この試合のハイライト、プレイヤーの焦りや気迫の変化、印象的な出来事をJSONで抽出せよ」と指示する。
   - _抽出例_: {"key_event": "豊中アリーナでの死闘", "player_sentiment": "終盤で声の震えが消え、気迫スコア0.9を記録", "critical_damage": "右腕に集中砲火"}
3. **Firestoreへの永続化**:
   - 抽出したハイライトを users/{userId}/matchLogs コレクションに新規ドキュメントとして追加する。
   - 同時に、機体の長期記憶サマリーである aiMemorySummary ドキュメントを`gemini-3.1-pro-preview`に書き直させ、一度だけ更新（Update）する。

### **3\. 戦歴を刻むプロシージャル・テクスチャ（視覚的進化）**

「レベルアップ」の報酬として、あらかじめ用意された画像ではなく、AIがその機体専用のテクスチャを生成・上書きするシステム。

#### **3.1 マイルストーン・トリガーの発火**

1. 試合数が特定の節目（例：10試合完了時、または勝率が急上昇した時）に到達したことをFirestoreのトリガー（Cloud Functions等）が検知する。
2. バックエンドの非同期バッチ処理として、テクスチャ生成タスクがキューイングされる。

#### **3.2 現実融合（8-Image Mix）によるテクスチャ生成**

1. **APIコール**: Nano Banana Pro (`gemini-3.1-flash-image-preview`) の画像融合APIを呼び出す。
2. **マルチモーダル・プロンプトの構成**:
   - **ベース画像**: 現在の機体のテクスチャ画像（例：木材メーカーの知識を活かしてプレイヤーが初期設定した「美しい無垢材のテクスチャ」）。
   - **追加画像**: 観客（娘さんたち）が過去の試合で投下したアイテムの画像（例：「春巻き」の画像）。
   - **テキストプロンプト**: aiMemorySummary から抽出した記憶を元に生成。
     - _"この木材パーツの表面に、激しい炎の攻撃を受けた黒い焦げ跡と、幾多の戦いを潜り抜けた細かい刃こぼれの傷をリアルに合成して。また、右肩には勝利のシンボルとして入力画像の『春巻き』をステンシル風のペイントロゴにして刻み込んで。"_
3. **動的適用（Server-Driven Rendering）**:
   - 1〜2秒で生成された新しいテクスチャURLがFirestoreに保存される。
   - 次回AR空間に機体を呼び出した際、Three.jsがこのテクスチャを読み込み、機体は「激戦の焦げ跡と春巻きロゴが刻まれた木製ロボット」へとシームレスに姿を変える。

### **4\. アンビエントな成長フィードバック（精神的進化）**

戦闘中ではなく、リラックスした「散歩モード（Phase 2）」中などに、AIが自律的に過去を振り返るエモーショナルなUX設計。

#### **4.1 MCP（Model Context Protocol）による自律的記憶検索**

1. **Visionベースの環境認識**: 散歩モード中、Gemini Live API（`gemini-2.5-flash-native-audio-preview-12-2025`）がスマホカメラの映像を監視。「夕日」や「見覚えのある手作りの木製家具（DIY作品）」といった特定のコンテキストを検知する。
2. **MCPサーバー連携**:
   - AIは自身に付与された Managed MCP servers for Firestore の権限を使い、自律的に自身の aiMemorySummary を検索しにいく。
3. **文脈の結合とProactive Audio（自発的発話）**:
   - AIが「現在の目の前の風景」と「過去の記憶」をリンクさせ、`gemini-2.5-flash-preview-tts`を通じて静かなトーン（tone: "subtle_reflection"）でプレイヤーに語りかける。
   - _発話例_: 「……マスター。この自作の木製テーブルの木目、綺麗だな。そういや、こないだ豊中のジイジの家で対戦した時、あんた焦って声が裏返ってたっけ（笑）。……でもあの時、最後は腹くくってバシッと詠唱決めてくれたよな。俺たち、ちょっとは良いコンビになってきたんじゃないか？」

### **5\. マイルストーンのエンタメ拡張（Veo / Lyria 連携）**

数十試合に一度など、長期的なリテンション（継続率）を高めるための特大の報酬システム。

#### **5.1 勝利のプロシージャル凱歌 (Lyria API)**

- 激闘を制して大きな大会（または連勝）をクリアした際、バックエンドで Lyria API が非同期起動する。
- 試合のハイライト（春巻きソードでの逆転劇など）を歌詞に組み込んだ、30秒の「少しコミカルで熱いEDM」を自動生成し、リザルト画面のBGMとして再生する。

#### **5.2 メモリアル動画生成 (Veo 3.1 API)**

- 50試合消化などの大きな節目で、Veo 3.1 API が稼働。
- これまで蓄積されたARプレイ中の録画データ（数秒のクリップ群）とAIの記憶を繋ぎ合わせ、「機体とマスターの軌跡」を描く高品質な15秒のショートムービーを生成。SNS（共有機能）への導線としてプレイヤーの推し活を爆発的に促進する。
