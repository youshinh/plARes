# 9\. マルチモーダル生成AI・拡張パイプライン設計書

[English Version (EN)](../multimodal_craft.md)

**対象モジュール**: エンタメ＆拡張機能**目的**: 観客のスマホで撮影した現実のアイテム写真とテキストから、gemini-3.1-flash-image-preview (Imagen) を用いて1〜2秒で3Dテクスチャを動的生成・マッピングする「現実融合クラフト」の実装を定義します1。また、Veo 3.1や gemini-2.5-flash-preview-tts を用いたマイルストーン動画や凱歌の非同期バッチ生成（イベント駆動型のダイナミック・ゲーティング）を設計します1。

## 1\. Nano Banana Proによる「現実融合クラフト」アーキテクチャ

観客の介入システムを「テキスト入力」から「現実世界の写真取り込み（Image Fusion）」へアップグレードし、日常のアイテムを3D空間の兵器や防具に変換するシステムです2。

### 1.1 観客介入（プロンプト・ドロップ）の処理フロー

- **マルチモーダル入力**: 観客（例：娘さんたち）がスマホのカメラでテーブルにある現実のアイテム（例：「揚げ春巻き」）を撮影し、チャット欄に付加したい概念のテキスト（例：「レーザーを放つ伝説の聖剣」）を入力して送信します2。
- **画像融合（8-Image Mix）の実行**: バックエンドのAPIが、送信された写真（Reference Image）のパリパリとした質感・形状と、テキストプロンプトを高度に融合（Image Fusion）させます2。
- **動的マッピング**: わずか1〜2秒で「揚げ春巻きの質感を残しつつ、黄金のレーザーを放つ巨大な聖剣」のテクスチャおよび3Dオブジェクトが生成され、アリーナ（AR空間）へ即座に乱入（投下）するロジックを実装します2。

### 1.2 3Dプリンター＆木工作品へのAIテクスチャ・マッピング

- プレイヤーが3Dプリンターで出力した無地のロボットパーツや自作の木工の盾をカメラで読み込ませ、「サイバーパンク風の塗装を施して」と指示することで、現実の物理的な形状はそのままに、AR空間上でのみ超絶クオリティの塗装（テクスチャ）がマッピングされる機能を実装します2。

## 2\. 戦歴を刻むプロシージャル・テクスチャ進化

機体のパーツに、データベースに蓄積された「長期記憶（Memory Bank）」と画像生成AIを連動させ、戦歴をテクスチャ（傷、汚れ、焦げ跡など）として刻むアーキテクチャです3。

### 2.1 Memory Bankと画像融合の連動

- 特定の節目（例：10試合完了時や強敵への勝利時）をトリガーとして、バックエンドの非同期バッチ処理を起動します4。
- Firestoreの aiMemorySummary から抽出した戦歴（例：「激しい炎の攻撃を受けた黒い焦げ跡」）に基づくテキストプロンプトと、現在の機体のベーステクスチャ（または特定のパーツのUVマップ画像）をリファレンス画像として、マルチモーダル入力を構築しAPIへ渡します4。

### 2.2 Three.jsによる動的マッピングとカスタムシェーダー

- APIが現実の質感を維持したまま生成した「戦歴が刻まれた新しいテクスチャ画像」のURLを、Firestoreの機体データに上書き保存します4。
- フロントエンド（React / Three.js環境）では、更新されたテクスチャURLを TextureLoader を用いて非同期で読み込み、機体モデルのマテリアル（mesh.material.map など）にアタッチし、material.needsUpdate \= true を発行して外見を即座に更新します5。
- さらに、Three.jsのカスタムシェーダー（WebGL）を用いて、ノーマルマップ（法線マップ）やラフネス（粗さ）の数値を戦績に応じて動的に書き換え、傷の深さ（凹凸）や光沢感をプログラマブルに制御する実装を行います5。

## 3\. Veo 3.1 / Lyria 連携によるイベント駆動型バッチ生成

計算コストと生成単価が高い動画・音楽生成AIは、常時稼働させるのではなく、プレイヤーの感情が最も高ぶる「マイルストーン」に限定して非同期で実行するダイナミック・ゲーティング設計とします6。

### 3.1 Lyria APIによる「勝利のプロシージャル凱歌」

- **実装フロー**: 試合終了時、バックエンドで勝敗判定を行い「勝利（WIN）」だった場合のみ非同期タスクとして発火させます7。Firestoreの matchLogs に保存された試合中のハイライトイベント（例：「春巻きソードで逆転した泥仕合」）をGeminiが分析し、最適なプロンプト（例：「アップテンポで少しコミカルなEDM」）を生成して gemini-2.5-flash-preview-tts または相当するAPIに送信します7, 8。
- **UXの反映**: 数十秒で生成された15〜30秒の楽曲を、試合直後の「勝利者インタビュー」のバックグラウンドBGMやリザルト画面の専用エンディングテーマとして再生させます7-9。

### 3.2 Veo 3.1 APIによる「マイルストーン総集編」ハイライト映像

- **バッチ処理の起動**: Firestoreの users/{userId}/profile ドキュメント内の総試合数（totalMatches）カウンターを監視し、5試合ごと（totalMatches % 5 \== 0）の条件を満たした場合に、専用の生成キュー（Pub/Subなど）へメッセージを送信し、非同期バッチを起動します10。
- **マルチモーダル入力と生成**: 過去5試合分の aiMemorySummary および matchLogs のハイライトテキストと、フロントエンド（Three.js）側で要所にてキャプチャ・保存しておいた数枚のスクリーンショット画像をクラウドストレージからロードし、Veo 3.1 APIに渡して「シネマティックなハイライト映像」を生成させます8, 10, 11。
- **SNS最適化とバイラル拡散**: 生成された映像は、縦型（9:16）に自動クロップされ、AI実況のテロップ（自動字幕）が焼き付けられます10, 11。観客のスマホ画面などに「動画を共有する」ボタンを表示し、LINEの家族グループやTikTokといったSNSへ「#plARes #ハゲ春巻きの悲劇」などのハッシュタグ付きで直接シェアできるルーティングを実装し、ゲームのバイラル拡散（Fandom拡張）を狙います10, 11。
